# Executable Task Definition: P3-T03

## Task Identity

- **Task ID:** P3-T03
- **Phase:** Phase 3 — Implement Repository Enforcement
- **Title:** Verify repository-enforcement requirements
- **Task type:** enforcement verification and evidence-report task
- **Predecessor dependency:** P3-T02 must be completed by merged pull-request evidence.
- **Task-state authority:** GitHub merged pull-request evidence determines predecessor completion; P3-T02 must be merged before P3-T03 may become ready. GitHub open pull-request evidence determines whether a harness task is active or awaiting human review. The merged Phase 3 register and qualifying-gate evidence determine active-phase status. This P3-T03 definition governs P3-T03 scope, deliverable, acceptance criteria, validation, and stop condition. Current GitHub PR evidence controls over stale static task-status wording. A missing or invalid required task-definition field blocks execution. Exactly one harness task or planning PR may be executing or awaiting human review at a time.

## Purpose

Produce a repository-native evidence report that verifies the implemented repository-protection and pull-request/CI enforcement against approved requirements. This task measures and records compliance; it does not configure controls or authorize a phase closure.

## Allowed Paths

- `docs/development/harness/PHASE-3-ENFORCEMENT-VERIFICATION.md`

The task may inspect GitHub repository settings, branch-protection/ruleset state, workflow configuration, pull-request checks, and merged predecessor evidence in read-only mode.

## Forbidden Scope

No game code; product, architecture, graph-domain, evaluator, content, diagnostics, persistence, or UI work; GitHub settings changes; branch-protection changes; workflow, CI, script, or repository configuration changes; Phase 4 or later work; or changes outside the allowed report path are authorized.

## Required Deliverable

Create `docs/development/harness/PHASE-3-ENFORCEMENT-VERIFICATION.md`. The report must record each requirement as **Pass**, **Fail**, or **Blocked**, with current repository evidence, for at least:

1. protected default branch and prohibition of direct pushes;
2. pull-request requirement and reviewability;
3. human approval and stale-approval dismissal;
4. required enforcement checks and their observed pull-request evidence;
5. branch freshness or approved merge-queue/update policy;
6. review-conversation resolution;
7. squash-merge policy and prohibited merge methods;
8. absence of harness branch-protection bypass;
9. CI validation of pull-request identity, scope, task dependencies, phase eligibility, authority evidence, required validation evidence, and applicable repository quality checks; and
10. evidence limitations, unavailable platform state, unverified items, and resulting risks.

The report must distinguish configured evidence from verified observed behavior. It must not claim that a requirement passes where evidence is unavailable.

## Acceptance Criteria

- The verification report exists at the allowed path and records every required category as Pass, Fail, or Blocked with evidence.
- The report distinguishes configuration inspection from observed workflow/check behavior.
- Every Fail or Blocked item identifies the exact unmet requirement or unavailable evidence and its consequence for Phase 3 closure.
- The report does not change repository controls, workflows, scripts, or implementation.
- No file outside the allowed path changes.

## Required Validation

- Inspect merged evidence for P3-T01 and P3-T02.
- Inspect the current `main` branch protection/ruleset configuration, applicable GitHub Actions workflow configuration, and pull-request/check evidence available to the task.
- Cross-check every report row against the two approved enforcement-requirement documents.
- Confirm only the allowed report path changed.
- Record unavailable GitHub settings/check evidence as Blocked, not Pass.
- Runtime game, product, architecture, and UI validation are intentionally not applicable and not authorized.

## Stop Condition

Open one reviewable P3-T03 pull request and stop. Await human review and merged GitHub evidence. Do not approve, merge, enable auto-merge, or claim completion before that evidence exists.

## Later-Task Boundary

This task cannot select, configure, draft, plan, or start P3-GATE-01, Phase 4, or any game, product, or architecture work.
